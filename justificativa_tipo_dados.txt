- Operacoes
	id (INT) - Int pois é o modelo padrão para indicação de id's em SQL
	usr_id (INT) - Int pois se refere ao campo id da tabela de Usuarios, logo teria que ser do mesmo tipo
	atv_id (INT) - Int pois se refere ao campo id da tabela de Ativos, logo teria que ser do mesmo tipo
	qtd (INT) - Int poi,s se trata de uma quantidade de operações sendo assim sempre numeros inteiros
	prc_unt (Decimal(18,2)) - Decimal é necessario pela precisção financeira, como representa o preço unitario do ativo no momento ta operação. E o (18,2) para ter representações bem precisas em relação ao valor.
	tipo (TINYINT) - Optei pelo uso do TINYINt por uma escolha efetuada durante a criação do projeto em C# por se tratar de dois valors fixos (Compra ou Venda) ficou mais simples trabalhar com um Enum e como o TINYINt representa apenas dois valores (0 e 1) foi extremamente preciso para o caso
	corretagem DECIMAL(10, 2) - DECIMAL pois se trata do valor final da corretagem pago na operação representando um valor monetario com duas casas decimais
	dt_hr DATETIME - DATETIME pois é a estrutura utilizada para trabalhar com datas e horas
